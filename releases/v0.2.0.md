# Autopraxis v0.2.0

Bundles the agent-fleet council/ship payload so `/council` and `/ship` install as native Autopraxis skills and agents.

## Highlights

- Vendored `@zhachory1/agent-fleet@0.4.0` under `vendor/agent-fleet/` (version-pinned).
- Skill and plugin installs now place `council`/`ship` skills plus council personas and ship agents alongside Autopraxis skills — no separate agent-fleet install step.
- Added `npm run sync:agent-fleet` to re-vendor the payload after a dependency bump, with a `validate-package` guard against version drift.

## Install behavior

- plugin-root targets (`claude-plugin`, `codex-plugin`, `mewrite-plugin`): `skills/council`, `skills/ship`, and `agents/*.md` personas + ship agents under the plugin root.
- skill-directory targets (`claude-skills`, `codex-skills`, `mewrite-skills`, `opencode-skills`): `council/` and `ship/` skill dirs with personas + ship agents in a sibling `agents/` dir.
- markdown-bundle targets (`generic-markdown`, `cursor-rules`, `windsurf-rules`): Autopraxis skills only; council/ship are not bundled for these flat layouts.

## Validation

Validated for this release:

```bash
npm test
npm pack --dry-run
npm exec -- agent-fleet --version
```

## Known Limitations

- Runtime council gates still require agent-fleet `lib/` helpers resolvable via `AGENT_FLEET_HOME`; bundling ships the helpers but does not change the preflight requirement.
- Markdown-bundle targets do not receive council/ship because flat layouts have no skills/agents directories.
